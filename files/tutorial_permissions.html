Permissions and ownership

The ls command along with its -l (for long listing) option will show you metadata about your Linux files, including the permissions set on the file. 

$ ls -l
    rwxr-xr-x. 4 root root    68 Jun 13 20:25 tuned
    -rw-r--r--. 1 root root  4017 Feb 24  2022 vimrc

In the context of a computer operating system, permissions refer to the rules or settings that determine who can access, modify, or execute files and directories. Permissions are a crucial aspect of system security, helping to control and restrict access to sensitive information or system resources. In Linux and Unix-like operating systems, including macOS, permissions are associated with users, groups, and others.

Here are the key concepts related to permissions:

a). Users:

1. Owner (User): The user who owns the file or directory. The owner has specific permissions.

2. Group: A collection of users who share common permissions on files or directories.

Permission Types:

1. Read (r): Allows the reading of file contents or listing directory contents.

2. Write (w): Allows the modification of file contents or the creation, deletion, and renaming of files within a directory.

3. Execute (x): Allows the execution of files or the traversal of directories.

b). Permission Levels:

1. Owner Permissions:Apply to the user who owns the file or directory.

2. Group Permissions:Apply to the group associated with the file or directory.

3. Others Permissions:Apply to users who are neither the owner nor part of the group.

c). Permission Representation:

ssions are often represented using a 10-character string. The first character indicates the file type, and the next three sets of three characters represent the permissions for the owner, group, and others, respectively.

In this example, the owner has read, write, and execute permissions; the group has read and execute permissions; and others have only read permissions.

d). Numeric Representation:

Permissions can also be represented using numeric values (octal representation).

Each permission type is assigned a numeric value: read (4), write (2), and execute (1).

The sum of these values represents the permission set.

-rwxr-xr--  (Symbolic representation)
    764         (Numeric representation)

In this example, the numeric representation for the permissions is 764.

e). Changing Permissions:

The chmod command is used to change permissions. For example:

chmod 755 filename

This command sets read, write, and execute permissions for the owner, and read and execute permissions for the group and others.

Special Permissions

Special permissions in Linux refer to advanced settings that can be applied to files and directories, in addition to the standard read, write, and execute permissions. These special permissions include the setuid (SUID), setgid (SGID), and sticky bit. These permissions provide specific functionality and control over how programs run and how directories are accessed.

a). Setuid (SUID):

When the setuid permission is set on an executable file, the program will run with the effective user ID (UID) of the file owner rather than the user who executes the program.

This is often used for programs that need elevated privileges temporarily. For example, the passwd command needs to update the password file, which regular users don't have direct access to. By setting the setuid bit on passwd, it runs with the elevated privileges required to update the password file.

Example:

chmod u+s /path/to/executable

b). Setgid (SGID):

When the setgid permission is set on an executable file or a directory, the program or any new files and directories created within that directory will inherit the group ownership of the parent directory rather than the group of the user who created them.

This is commonly used for shared directories where multiple users need access to the same files with a common group ownership.

Example:

 chmod g+s /path/to/executable

c). Sticky Bit:

The sticky bit, when applied to a directory, restricts the ability to delete or rename files within that directory to the file owner, the directory owner, or the superuser. It is often used on directories where multiple users have write access to prevent users from deleting each other's files.

Example:

 chmod +t /path/to/directory

The sticky bit is typically set on directories like /tmp, allowing any user to write to the directory but preventing them from deleting or modifying files created by other users.
